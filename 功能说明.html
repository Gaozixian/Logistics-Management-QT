🚚 QT物流管理系统

基于Qt框架开发的现代化物流管理解决方案，提供高效、可靠的订单管理与追踪功能

核心功能概述

这是一个基于 Qt框架 开发的物流管理桌面应用，使用 SQLite 嵌入式数据库存储数据，主要功能包括：

📦

订单管理

完整的增删改查功能，支持多条件筛选和排序

📑

多标签页界面

订单管理/新建订单/订单操作分页处理，提高工作效率

💾

数据库管理

支持数据库文件自由切换，自动检测表结构完整性

📊

可视化表格

使用QTableView展示数据，支持自定义列和排序

🔄

状态流转

订单状态自动流转：待发货→运输中→已送达

详细功能说明

功能模块 实现细节
订单管理
  • 表格展示所有订单（ID/客户/目的地/重量/状态等）
  • 支持点击查看详情
  • 自动刷新数据
订单搜索
  • 支持按客户名、目的地、订单号模糊搜索
  • 实时刷新搜索结果
新建订单
  • 表单验证（必填字段检查）
  • 重量/体积数值输入控件（QDoubleSpinBox）
订单操作
  • 删除订单（二次确认）
  • 状态更新（三态自动流转）
数据库管理
  • 默认使用logistics.db
  • 支持打开外部SQLite文件
  • 自动检测表结构完整性
用户界面
  • 响应式布局（QGridLayout/QHBoxLayout/QVBoxLayout）
  • 状态栏显示操作反馈

技术实现亮点

数据持久化

  • 使用SQLite嵌入式数据库，单文件便携存储
  • 预处理SQL语句防止注入攻击
query.prepare("INSERT INTO orders (...) VALUES (?, ?, ?)");

模型-视图架构

  • QStandardItemModel + QTableView 实现数据绑定
  • 自定义refreshOrderTable()保持数据一致性

交互设计

  • 删除订单前弹出确认对话框
    • QMessageBox::question(this, "确认删除", "确定删除订单吗？");
  • 状态变更后自动更新时间戳

异常处理

  • 数据库错误捕获并友好提示
    • QMessageBox::critical(this, "错误", db.lastError().text());

运行要求

  • 开发环境: Qt 5.15+ / Qt 6.x
  • 编译工具: MinGW/MSVC
  • 依赖库: QtSql、QtWidgets
  • 部署方式: 单可执行文件+数据库文件

使用示例

1. 启动程序 → 自动加载logistics_default.db 2. [新建订单] 填写客户/目的地信息 → 点击"添加订单" 3. [订单管理] 双击表格行查看详情 → 点击"更新状态" 4. [文件菜单] 可切换其他数据库文件
